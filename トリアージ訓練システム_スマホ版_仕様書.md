# トリアージ訓練システム スマホ版 仕様書

## 0. この文書について

本書は「トリアージ訓練システム スマホ版」（`トリアージ訓練システム_スマホ版.html`）の仕様書である。スマホ版は PC 統合版（`トリアージ訓練システム_統合版.html`）のサブセットであり、**シンポジウム向けのデータ収集**を主目的として、タイムアタック・練習モードの2モード／Stage 1・2 のみに機能を絞った派生版である。

START法・PAT法の判定ロジック、傷病者データ、スコアリングの詳細は既存仕様書を正とする：

- `STARTトリアージゲーム_仕様書.md`（START法・1次）
- `PAT法2次トリアージ_仕様書.md`（PAT法・2次）

本書では **スマホ版固有の仕様（統合版との差分）と、データ収集の互換性担保** を中心に記述する。

- 対象環境: スマートフォンブラウザ（iOS Safari / Android Chrome 最新版）。PCブラウザでも動作する。
- 配布形態: **単一HTMLファイル**（HTML / CSS / JavaScript / 傷病者データをすべて1ファイルに内包）。外部JSファイル（`jsファイル保管場所/`）への参照なし。
- 外部依存: なし（フォントはシステムフォント）。**結果送信時のみ**ネット接続が必要（Google Apps Script へ POST）。それ以外は完全オフライン動作。
- 言語: 日本語UI。
- バージョン: スマホ版 Ver.1.0（2026-06-12）

---

## 1. 目的

### 1.1 シンポジウム向けデータ収集
タイムアタックの成績（正答率・アンダートリアージ率・平均判定時間など）を、統合版と**同一の Google スプレッドシート**に収集し、訓練効果の検証データとしてシンポジウム発表に用いる。

- 参加者が自分のスマホで実施できるため、PC設置数に制約されず多人数からデータを集められる
- 実施者名＋資格＋消防歴＋救急歴を毎回同一入力してもらうことで、**同一人物の反復効果**を分析できる

### 1.2 学習目的
統合版と同じ（START法／PAT法の判断フローの反復学習）。練習モードは判定後にフロー解説を表示する初学者向け。練習モードの結果はデータ送信対象外。

## 2. 統合版との機能差分

| 機能 | 統合版 | スマホ版 |
|---|---|---|
| START法 タイムアタック | ○ | **○** |
| START法 練習モード | ○ | **○** |
| PAT法 タイムアタック | ○ | **○** |
| PAT法 練習モード | ○ | **○** |
| ステージクリアモード | ○ | ×（削除） |
| 記載訓練モード（読み上げ） | ○ | ×（削除。VOICEVOX音声・合成音声も不使用） |
| 1次→2次 連携訓練 | ○ | ×（削除） |
| Stage 1（雑踏事故） | ○ | **○** |
| Stage 2（交通多重事故） | ○ | **○** |
| Stage 3〜5 | ○ | ×（削除） |
| 結果送信（GAS→スプレッドシート） | ○ | **○（同一URL・同一ペイロード）** |
| CSV出力 | ○ | **○** |
| 印刷/PDF保存 | ○ | **○** |
| 成績履歴（localStorage・過去平均表示） | ○ | **○（同一キーで互換）** |
| 自己診断テスト（judge() テスト） | ○ | ×（削除） |
| キーボードショートカット（1〜4キー） | ○ | ×（タッチ操作のみ） |
| 音声フィードバック（正解/不正解音） | ○ | ○ |

### 2.1 ロジック・データの同一性（重要）
収集データを統合版と直接比較できるよう、以下は統合版から**無改変で内蔵**している：

- `CFG` / `PAT_CFG`（閾値・配点・制限時間の定数）
- `judge()` / `judge2()` / `triageStep()` ほか判定ロジック一式
- 傷病者データ `PT`（**全44名**）・`SECONDARY`・`HX`・`PREG`

ステージ定義 `STAGES` のみ Stage 1・2 の2件に制限している（傷病者IDの構成は統合版の Stage 1・2 と同一）。

> **傷病者を全44名内蔵する理由**: Stage 1・2 で使う傷病者だけに削ると配列インデックス（＝画面表示の No.／送信レコードの patient 対応）が統合版とずれる。No. と正解タグの完全一致を保つため、データは削らない。

### 2.2 スマホ版のメンテナンス上の注意
スマホ版は単一ファイルにJSを内蔵しているため、統合版の `jsファイル保管場所/*.js` を修正した場合、**スマホ版には自動反映されない**。傷病者データ・判定ロジック・送信仕様を変更した際は、スマホ版にも同じ修正を手動で反映すること（特にバイタル値や正解タグが変わる修正は、データ比較が壊れるため必須）。

## 3. ゲーム仕様

### 3.1 モード
- **タイムアタック**: 制限時間内に何人判定できるか。傷病者はステージ内をシャッフルし、使い切ったら再シャッフルして無限供給（直前の傷病者が連続しないようガード）。
  - START法: **180秒**（`CFG.TIME_ATTACK_SEC`）
  - PAT法: **240秒**（`CFG.TIME_ATTACK_PAT_SEC`）
  - 残り30秒で時計が赤色点滅
- **練習モード**: 制限時間なし（タイマー表示は `--:--`）。判定ごとに START フロー／PAT 段階の解説オーバーレイを表示し、「次の傷病者へ」で進む。ステージ全員（START: 全員、PAT: 黒タグ除く）を処理したら結果画面へ。

### 3.2 ステージ
| ステージ | 出題内容 | START出題数 | PAT出題数（黒除外） |
|---|---|---|---|
| Stage 1｜雑踏事故 | ids: 0,1,2,3,4,6,7,8,9,10,11,13,14,15,29 | 15名 | 14名 |
| Stage 2｜交通多重事故 | ids: 0,1,2,6,7,8,9,13,14,15,16,20,21,22,25,26,27,29,30,23 | 20名 | 18名 |

PAT法では `secondary === null` の傷病者（黒タグ：id 29〜32）を出題から除外する（統合版と同じ）。

### 3.3 スコアリング（統合版と同一）
- 正解: +100点 ＋ 速度ボーナス `max(0, 50 − 経過秒×5)`
- 5連続正解ごとにコンボ倍率 +0.1（最大 ×2.0）
- アンダートリアージ: −150点 ／ オーバートリアージ: −30点 ／ 赤↔黒取り違え: −200点

### 3.4 タグ選択
- START法: 赤Ⅰ／黄Ⅱ／緑Ⅲ／黒0 の4ボタン（2×2グリッド）
- PAT法: 赤Ⅰ／黄Ⅱ／緑Ⅲ の3ボタン（黒は2次対象外のため非表示。統合版と同じ）

## 4. データ送信仕様（シンポジウム向け収集）

### 4.1 送信先・方式
- 送信先: 統合版 `results.js` と**同一の GAS ウェブアプリURL**（`SHEET_WEBAPP_URL` 定数）。スマホ版・統合版の結果は同じスプレッドシートに集約される。
- 方式: `fetch` POST、`mode:'no-cors'`、`Content-Type: text/plain;charset=utf-8`（CORSプリフライト回避。統合版と同一）
- 重複排除: サーバ側で `sessionId` により実施。クライアント側でも送信済みセッションは再送ボタンを無効化。
- GAS側の設定手順: `結果送信_Googleスプレッドシート設定手順.md` 参照。**GAS・スプレッドシート側の変更は不要**（送信スキーマが同一のため）。

### 4.2 送信対象
- **START法・PAT法のタイムアタックのみ**（`canSendResult()`）。練習モードの結果画面には送信ボックス自体を表示しない。

### 4.3 送信ペイロード（統合版と同一スキーマ）
セッションサマリ: `sessionId / sentAt / participant / qual / fireYears / emsYears / deviceId / triage / mode / stage / title / score / total / correct / accuracy / under / over / underRate / overRate / blackRed / avgSec / maxCombo / weak`

明細 `records[]`（1判定=1レコード）: `no / phase / patient / age / step / chosen / correct / result / errorType / elapsedSec / points / at`

- `triage`: `START(1次)` または `PAT(2次)`
- `stage`: `Stage 1｜雑踏事故` / `Stage 2｜交通多重事故`（統合版と同一表記）
- スマホ版であることを示す専用フィールドは**追加しない**（スキーマ同一性を優先）。端末判別が必要な場合は `deviceId` とユーザーエージェント傾向で行う。

### 4.4 実施者プロファイル
- 入力項目: 実施者名（必須・30字以内）、資格（救急救命士／標準課程／その他）、消防歴・救急歴（０～５年未満／５年～10年未満／10年以上）
- `localStorage` キー `triageParticipantProfile` に保存し、次回以降は自動入力（統合版と同一キー）
- 画面注記:「同じ人は毎回同じ名前・資格・歴で送信してください（反復効果の分析に使用）」

### 4.5 運用上の注意
- 送信にはネット接続が必要。訓練会場では Wi-Fi またはモバイル回線を確保すること。
- 送信失敗時はエラーメッセージを表示し、ボタンを再度有効化（再送可能）。
- 本番運用前に、各自のスマホで1件テスト送信し、スプレッドシートへの記録を確認すること（テスト行は分析時に除外）。

## 5. 画面・UI仕様（スマホ最適化）

### 5.1 画面一覧
1. **タイトル画面**: 方式×モード選択（START/PAT × タイムアタック/練習の4ボタン）、Stage 1/2 選択、音声フィードバックON/OFF、データ収集協力依頼文、ゲーム開始、START法確認／PAT法確認
2. **ルール画面（START法／PAT法）**: フロー図・区分表・スコア表（統合版と同内容）
3. **ゲーム画面**: 上部固定ヘッダー（中断✕／ステージ名／タイマー／コンボ／スコア／進捗）＋傷病者カード＋下部固定タグボタン
4. **結果画面**: スコア6指標カード／評価・診断（過去平均比較）／苦手ステップ表／誤判定レビュー（最大8件＋解説）／結果送信ボックス／CSV出力・印刷/PDF保存・タイトルへ・もう一度

### 5.2 モバイル最適化
- viewport: `width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no`（誤ピンチズーム防止）
- `touch-action: manipulation`（ダブルタップズームの300ms遅延・誤拡大防止）
- `-webkit-tap-highlight-color: transparent`、ボタンは `user-select: none`
- **タグボタンは `position:fixed` で画面下部に常時固定**（スクロール不要で親指リーチ内）。START: 2×2グリッド、PAT: 3列。最小高さ62px。
- セーフエリア対応: `env(safe-area-inset-top / bottom)`（ノッチ・ホームバー回避）
- ゲーム本文は下部ボタンに隠れないよう `padding-bottom: 190px`
- 傷病者切替時に自動で先頭へスクロール
- 結果画面の送信フォーム・評価グリッドは縦1カラム（480px以上で多カラムに切替）
- 想定最小幅 375px（iPhone SE 相当）。380px未満では所見グリッドが1カラムに縮退

### 5.3 配布方法
単一ファイルのため以下のいずれでも配布可能：
- Webサーバ（GitHub Pages 等）に置き、URLのQRコードを会場掲示 ← **シンポジウム運用の推奨**
- LINE・メール等でHTMLファイルを直接送付（ファイルを開くだけで動作）
- ※ `file://` で開いた場合も結果送信は動作する（no-cors POST のため）
